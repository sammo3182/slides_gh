1.1 面向对象编程

  • 面向对象编程（Object-Oriented Programming, OOP）：通过类与对象组织、封装数据
  • C++、Java、Python、R 都采用 OOP 架构
  • 与面向过程编程（POP）相比，OOP 更擅长处理复杂系统和多变需求
  • 两个基本概念：类（class）与对象（object）
    • 类：对一组对象的总体规定（如”马”、“主食”）
    • 对象：类的具体实例（如”白马”、“土豆”）
    • 属性（property）：对象的特征
    • 方法（method）：可作用于该类对象的动作
  • OOP 的好处：先归类、再批量操作，避免重复编程；例：脏上衣、脏外裤 → “脏衣物”类 → 统一”放进洗衣机”

1.2 函数：状语后置的祈使句

  • 函数（function）：状语后置的祈使句
  • 数学含义：y = f(x) 里的 f
  • R 中函数 = 数学映射 = 一组可执行动作
  • 已见：c()、matrix()、mean()
  • R 函数语法：

<结果对象名> <- <函数>(<目标数据>, <条件1>, <条件2>, ...)

  • 含义：基于条件对目标数据进行函数操作，结果存到对象名
  • 等价于英语”状语后置的祈使句”
  • 自定义函数：function(<数据们>){<动作们>}
  • 示例：定义 BMI 计算函数（BMI = 体重 / 身高²）

自动”打印”

  • 直接敲函数名会触发 auto-print：结果显示在控制台
  • 优点：即刻看到结果
  • 缺点：结果不被保存，后续无法调取
  • 函数命名规则：
    • 必须以字母开头
    • 仅限字母、数字、. 和 _
    • 区分大小写

命名规则的由来

  • 字母开头：让计算机区分”名字”与”数字”
  • 不含特殊字符：避免与编程符号冲突
  • 大小写敏感：精确性要求
  • 命名习惯：
    • 动词或动词短语：read_rds、saveRDS
    • 名词：lm（linear model）、dotwhisker
    • 常见缩写：sqrt（square root）、exp（exponential）
  • 名称要表意：让看代码的人能猜出函数功能
  • 避免覆写 R 内置函数（如 mean、list、sum）

1.3 软件包：函数的集合

  • 软件包（package）：函数的集合
  • 类比：装着各种工具的”工具箱”
  • 获取途径：
    • CRAN（综合 R 档案网络）：经审核、有规范保证
    • GitHub 等开源平台：免审核、含开发版
  • 加载软件包：library() 或 require()
  • require() 多一步检查包是否已加载，返回逻辑值
  • 安装与加载示例：
# 从 CRAN 安装包（只需做一次）
install.packages("drhur")

# 从 GitHub 安装包（开发者版本）
remotes::install_github("sammo3182/drhur")

# 加载包（每次重启 R 都要做）
library(drhur)

# 调用包内的函数打开课件
drhur("play")
  • 帮助文档调取方式：
    • ?函数名：例 ?install.packages
    • help(函数名)：等价用法
    • help.search("关键词")：搜索
    • help.start()：打开总文档（不常用）

安装 R 包的预期时间

  • 一般几秒到十几秒
  • 第一次安装会连带装依赖包，耗时更长
  • 满屏滚动属于正常现象，静待结束即可

1.4 OOP 编程基本性质

  • 架构在 OOP 上，R 具有四大特性：
    • 继承（inheritance）
    • 封装（encapsulation）
    • 多态（polymorphism）
    • 安全（safety）
  • 继承性：子类可继承母类的全部特性
  • 用 inherits(对象, "类名") 检验
  • 例：ordered() 创建的定序变量继承自 factor
  • 封装性：把一系列命令打包成一个命令组合
  • 调用打包后的命令，R 会按流程跑完所有既定步骤
  • function() 就是常用的封装命令

封装的可覆写性

  • R 中函数也是对象，“万物皆可函数”
  • 已定义的函数可被覆写，包括内置函数
  • 危险示例：mean <- function(x){x^2} 会让所有 mean() 变为平方运算
  • 请谨慎覆盖、修改、删除内置命令
  • 多态性：同一函数对不同对象有不同处理
  • 典型例子：summary()——因材施教
  • 安全性：命令作用前先判断对象类的兼容性
  • 不兼容则停止运行并报错，后续命令不再执行

命令、方法、函数

  • 命令（Command）：任意一行可执行代码（如 1 + 1、y <- sqrt(x)）
  • 函数（Function）：一套固定操作，接收输入并返回结果
  • 方法（Method）：针对特定类对象的函数
  • OOP 基本原则：不直接触碰原始数据
  • 数据处理采用”对象+命令”方式
  • 优点：每步可回溯、安全性高、便于协作

1.5 赋值（创建）

  • R 中创建数据 = 将现实观察转换为可分析的数据形式
  • R 按类（class）分类记录，每类有各自属性与方法
  • 常见类：向量（vector）、数据框（data frame）、列表（list）
  • 赋值（assign）：给值或对象起个名字、缓存以备调用
  • 最常用赋值符：<-
  • 同效命令：assign()、->、<<-、=
  • 基础语法：<名称> <- <对象>
  • <- 与 -> 方向相反，箭头所指的一方就是被赋值的名称
  • = 也可赋值，与 <- 略有不同：
    • 顶层环境：二者作用域相同
    • 函数调用括号内：= 多用于参数命名，<- 多用于创建/修改对象
  • <<- 用来援引/修改父级（全局）变量
  • 在多重环境嵌套时使用，区分作用域
  • 示例：校园公告栏（全局）vs 个人提醒（局部）
  • 对象命名规则：
    • 以字母或 . 开头（若以 . 开头，下一位不能是数字）
    • 不能以数字开头
    • 除 . 与 _ 外，不能含其他特殊符号
    • 区分大小写：X != x
    • 谨防覆写内置命令（如 list、mean）

1.6 向量

  • 向量（vector）：R 中所有”一串”数据的统称
  • 用 c() 或 vector() 创建
  • 严格同质：一个向量内的元素必须同一类型
  • 子类：数值型（numeric）、字符型（character）、逻辑型（logical）、类别型（factor）、时间（POSIXct/POSIXlt）
  • 数值向量（numeric vector）：可代数运算的数值
  • 子类型：双精度（double，含小数）、整数型（integer）
  • NA（not available）：缺失值，是 R 同质向量的”例外”
  • 逻辑向量（logic vector）：二元变量 TRUE/FALSE（可简写 T/F）
  • 应用极广：很多”编程”本质就是”判断真假”
  • as.logical() 强制转换为逻辑型
  • 类别向量（factor vector）：记录分类信息
  • factor() 创建，水平（level）记录类别，标签（label）给水平起名
  • 子类：定序向量（ordinal vector），ordered() 创建，水平含顺序
  • 进阶工具：forcats 包专门处理分类/定序变量（改水平顺序、添加类别、调整参照组）
  • 字符向量（character vector）：记录字符形式的值
  • R 不”读懂”字符含义，对中、英、俄等都一视同仁
  • 创建时每个字符值外需加引号
  • 预存字符常量：letters、LETTERS、month.abb、month.name
  • 时间向量：R 提供 POSIXct 与 POSIXlt 两种
  • POSIXct：以秒数（自 1970-01-01 起）存储
  • POSIXlt：以列表形式存储时间各部分
  • 当前时间：Sys.time()、Sys.Date()、Sys.timezone()

lubridate 包

  • 专门处理时间变量
  • 灵活记录年月日排列、时区转换等

1.7 数据框与列表

  • 数据框（data frame）：列等长的二维结构，列可异质
  • 新文科实践中”数据”通常指数据框
  • 提取方式：
    • 坐标法：df[行, 列]
    • 名称法：df$列名（最常用）
  • 列表（list）：异质容器，存放一串向量
  • 各向量的大小、长度、顺序无限制
  • 数据框 = 列等长、各列一一对应的特殊列表

多维数据的阵列 (array)

  • 用 array() 创建，多个维度各保存一个向量
  • 社科人文领域使用机会不多

1.8 数据属性

  • 探针型工具：查询对象属性
    • class() / typeof()：变量属性 / 内存存储方式
    • nchar()：字符串长度
    • levels()：因子水平
    • length()：长度（向量看元素、列表看组件、数据框看列数）
    • nrow() / ncol() / dim()：维度
    • summary()：统计信息
  • class()：R 把对象”看作”什么 → 决定可用哪些函数
  • typeof()：对象在内存里的底层存储方式
  • 数据框的 length() 返回列数（因为本质是列表）
  • 查行数用 nrow()、列数用 ncol()、全维度用 dim()
  • 判断型工具：通常以 is 开头（is.numeric()、is.character()、is.factor()、is.data.frame()、is.list()），返回逻辑值表示对象是否属于该类型

1.9 内置数据

  • 实际工作中手动创建数据机会不多，多数情况是从外部导入
  • 三种数据输入方式：内置数据、直接读取、调用包读取
  • 内置数据（built-in data）：R 自带或软件包附带
  • 调取清单：data()
  • 调用单个数据集：data(<数据名>)
  • 调用包内数据集：data(<数据名>, package = "<包名>")
  • 调用其他包内的数据需指定 package 参数
# 尝试直接找 corruption（未加载 regioncode 包则失败）
data(corruption)

# 明确告诉 R 去 regioncode 包里找
data(corruption, package = "regioncode")

summary(corruption)
  • R 自带的 datasets 包：无需安装/加载，直接调用
  • 例：iris、mtcars、uspop 等

1.10 直接读取数据

  • 直接读取支持的数据类型：
    • .RDS：R 专有，单一对象
    • .RData：R 专有，可保存多个对象
    • .txt：带分隔符的表格
    • .csv：逗号分隔值
  • 输入语法：<名称> <- <读取函数>(<数据路径>)
  • 读取函数常以 read 开头
# 读取 RDS 格式
df_rds <- readRDS("aDataset.rds")

# 读取 TXT 格式 (通用表格读取)
df_txt <- read.table("D:/aDataset.txt")

# 读取 CSV 格式
df_csv <- read.csv("./aDataset.csv")

# 加载 RData (不需要建立对象)
load("aDataset.RData")

文件名大小写

  • Windows / Mac：.RData 与 .rdata 大小写不敏感
  • Linux：严格区分大小写，二者是不同文件
  • 建议：统一使用小写 .rds、.rdata

1.11 调用读取数据

  • 调用专门软件包读取其他格式
  • 常见包：
    • haven：SPSS / Stata / SAS
    • readr：CSV / TXT（速度更快）
    • readxl：Excel
    • rjson：JSON
    • XML：XML / HTML
# --- 统计软件 ---
library(haven)
df_spss <- read_spss("<FileName>.sav")
df_stata <- read_dta("<FileName>.dta")
df_sas <- read_sas("<FileName>.sas7bdat")

# --- 纯文本表格 ---
library(readr)
df_csv <- read_csv("<FileName>.csv")
df_table <- read_table("<FileName>.txt")

# --- Excel ---
library(readxl)
df_excel <- read_excel("<FileName>.xlsx")

# --- JSON ---
library(rjson)
df_json <- fromJSON(file = "<FileName>.json")

# --- XML / HTML ---
library(XML)
df_xml <- xmlTreeParse("<url>")
df_html <- readHTMLTable("<url>", which = 3)
  • “瑞士军刀”：rio 包的 import() 函数
  • 自动识别格式，一个函数读所有
  • 缺点：相对慢一些（二传手）
library(rio)
df_anything <- import("<AnyTypeOfData>")

1.12 储存为 R 数据

  • 数据输出基础语法：<函数名>(<储存对象>, <储存路径>, <其他条件>)
  • 函数名常以 save、write 开头
  • R 专有格式：
    • .rds：单一对象，用 saveRDS()
    • .RData：多个对象，用 save()
  • 推荐 .rds：调取和整理更清晰
  • .RData 的内容只能 load() 之后才能看到
# 保存为 RDS 单一对象
saveRDS(df_toy, file = "df_toy.rds")

# 保存为 RData 工作空间（可同时保存多个对象）
save(df_toy, ls_monks, file = "./data/test.rdata")

1.13 储存为 csv 文件

  • CSV：跨平台通用格式
  • 压缩效率不高，但几乎所有平台、所有程序都能读取
  • 用 write.csv() 保存
# 将 df_toy 保存为 CSV 文件
write.csv(df_toy, file = "toy.csv")

CSV 与中文数据

  • 中文数据存储时要选择 UTF-8 编码
  • 否则可能出现乱码现象

1.14 储存为其他格式

  • 用 rio 包的 export()
  • 自动识别后缀，将 R 对象存为几乎任何格式
  • 支持 STATA、SPSS、SAS、Excel、JSON、Matlab、HTML 等

数据存储效率

  • 中小规模数据：.RData 一般足够
  • 大体积数据：考虑 qs、fst 等特殊格式
  • 特殊格式兼容性较弱、存储稳定性有风险（换台电脑可能打不开）
