1 面向对象编程

  • OOP：通过数据组织和封装处理复杂系统
  • 两个基本概念：类（class）、对象（object）
  • 类比：脏衣物→洗衣机；书→书柜；电子产品→充电
  • 属性 = property；方法 = method
  • R 是基于泛型函数的 OOP 语言
  • 在介绍 OOP 特性前，先讲“函数”

2 函数：状语后置的祈使句

  • 函数 ≈ 数学里的 \(f\)，也像语言里的“动词”
  • 语法：<结果对象名> <- <函数>(<目标数据>, <条件1>, <条件2>, ...)
  • <- 表“将右边存到左边”
  • 函数括号里第一位通常是数据，其余为“状语”
  • 用 function(<数据们>){<动作们>} 自定义函数

自动“打印”

  • 不赋值时，函数结果会在控制台 auto-print
  • 好处：即刻看到结果
  • 坏处：未保存，无法在后续步骤调用
  • 函数命名规则：
    1. 必须以字母开头，不能以数字或 _ 开头
    2. 仅限字母、数字、.、_，不可含特殊符或空格
    3. 区分大小写（hey ≠ Hey ≠ HEY）

命名规则

  • 字母打头：避免和数字混淆
  • 排除特殊字符/空格：避免与运算符冲突
  • 区分大小写：计算机眼中两者完全不同
  • 常见命名形式：动词/动词短语；名词；常见缩写
  • 比如read_rds、saveRDS、lm、sqrt、exp

3 软件包：函数的集合

  • R 包 = 函数的集合（“工具箱”）
  • 两条获取途径：CRAN 与 GitHub 等开源平台
  • 加载用 library() 或 require()
  • install.packages()：从 CRAN 下载到本地（一次即可）
  • library()：每次重启都要重新加载
  • drhur("play")：打开本章课件
  • 调取帮助文档的常用方式：
    • ?install.packages
    • help(install.packages)
    • help.search("install.packages")
    • help.start()

安装R包的预期时间

  • 单包通常几秒到十几秒
  • 满屏滚动多为依赖包安装，属正常现象

4 OOP编程基本性质

  • R 的四大特性：继承性、封装性、多态性、安全性
  • 继承性：子类继承母类全部特性
  • 用 inherits(<对象>, "<类名>") 检验
  • ordered 继承自 factor，故 factor 工具大多直接可用
  • “认祖归宗”即可复用工具
  • 封装性：把一系列命令打包成一个组合命令
  • function() 即典型封装工具

封装的可覆写性

  • R 中函数也是对象（“万物皆可函数”）
  • 已有函数可被覆写，重启前生效
  • 谨慎覆盖 mean()、sqrt() 等内置函数
  • 多态性：同一函数对不同对象不同处理
  • 典型例子：summary()
  • 安全性：类不匹配则停止运行并报错

命令、方法、函数

  • 命令（Command）：任意可执行代码行
  • 函数（Function）：固定操作集，输入参数→返回结果
  • 方法（Method）：针对特定类对象的函数

5 记录

  • 不同类型数据按 class 分类记录

6 赋值（创建）

  • 赋值 = 给对象起名字
  • 主要赋值符：<-（推荐）、->、=、<<-、assign()
  • <- 与 ->：箭头所指方向为被赋值名
  • = vs <-：顶层等价；函数调用括号内 = 用于命名参数
  • <<-：跳出当前作用域，修改父级/全局变量
  • 命名注意事项：
    1. 字母或 . 开头；若 . 开头，下个字符不能是数字
    2. 不可以数字开头
    3. 仅限 . 与 _ 等少数特殊字符
    4. 区分大小写
    5. 谨防覆写内置命令

练习3：命名规则

解释为什么第一行会报错，第二行不会？

  • 回顾命名规则第二条
  • 看名称的首字符
  • R 解析器读到 5 时会怎么做？
# 5var_name <- 1:5  报错
# 原因：R 命名规则要求变量名以字母（或.）开头，不能以数字开头
# 解析器读到 5 时会期望一个数值表达式，遇到 var_name 就无法继续解析

var_name5 <- 1:5  # 合法：以字母开头，数字可出现在中间或结尾

7 向量

  • 向量 = 一串同类型数据
  • 用 c() 或 vector() 创建
  • 子类：数值（numeric）、字符（character）、逻辑（logical）等
  • 数字向量：可代数运算；分双精度型与整数型

缺失数据

  • NA = not available
  • 用以记录“本应有但缺失”的数据
  • 可由人为或前序计算造成
  • 逻辑向量：TRUE/FALSE 或 T/F
  • 数值与逻辑可互转：as.logical()、as.numeric()
  • 类别向量：factor() 创建；属性含 level、label
  • 定序向量：ordered()，水平含顺序

forcats包

  • 专门处理分类/定序变量
  • 改变水平顺序、添加类别、调整参照组
  • 字符向量：用引号标注
  • 内置字符常量：letters、LETTERS、month.abb、month.name

练习4：内置字符常量

比较两个命令生成的向量，思考它们的等价性与适用场景。

  • letters 是 R 预置的小写字母向量
  • 用 [1:5] 取前 5 个
  • 可用 identical() 验证两者是否相等
vec_letters_raw <- c("a", "b", "c", "d", "e")
vec_letters_quick <- letters[1:5]

# 验证两者是否完全相等
identical(vec_letters_raw, vec_letters_quick)

# 两者结果完全一样
# - vec_letters_raw：手动逐个输入，适合无规律字符
# - vec_letters_quick：利用预置常量，效率更高
  • 时间向量：POSIXct（秒数）/ POSIXlt（列表）
  • as.POSIXct()、as.POSIXlt() 转换
  • Sys.time()、Sys.Date()、Sys.timezone() 获取系统时间
  • 打印效果相同，底层存储天差地别（秒数 vs 列表）

lubridate包

  • 灵活记录各种时间格式
  • ymd/mdy/dmy 解析不同顺序
  • ymd_hms 含时分秒；可指定时区

8 数据框与列表

  • 二维结构化数据 → 数据框
  • 列可不同类型；可视为“变量的集合”
  • 提取：[行, 列] 或 $列名
  • 列表 = 一串向量的容器，长度/类型/顺序自由
  • 数据框是“等长列表”的特例

用于记录多维数据的阵列

  • 多维数据可用嵌套列表或阵列
  • array() 创建，常用于多维度参与运算

9 数据属性

  • 探针型工具：
    • class()、typeof()：变量属性
    • nchar()：字符串长度
    • levels()：因子水平
    • nrow()、ncol()、dim()：维度
    • summary()：统计摘要
  • class() 是“身份”，typeof() 是“底层存储”
  • length()：向量=元素数；列表/数据框=组件数（即列数）
  • 行数用 nrow()，列数用 ncol()，二者用 dim()
  • 判断型工具：is.* 系列
  • 例：is.numeric()、is.factor()、is.data.frame()、is.list()、is.matrix()

12 直接读取数据

  • 可直接读取：.RDS、.txt、.csv、.RData
  • 语法：<名称> <- <读取函数>(<数据路径>)

# 读取 RDS 格式
df_rds <- readRDS("aDataset.rds")
# 读取 TXT 格式 (通用表格读取)
df_txt <- read.table("D:/aDataset.txt")
# 读取 CSV 格式
df_csv <- read.csv("./aDataset.csv")
  • .RData 可保存多个对象，用 load() 读取
  • Linux 区分大小写；建议统一使用小写后缀

# 加载 RData 文件
load("aDataset.RData")

13 调用读取数据

  • 专门软件包：haven、readr、readxl、rjson、XML

# --- 导入统计软件数据 ---
library(haven) # 加载 `haven` 包
df_spss <- read_spss("<FileName>.sav") # 读取 SPSS 的数据文件 (.sav 格式)
df_stata <- read_dta("<FileName>.dta") # 读取 Stata 的数据文件 (.dta 格式)
df_sas <- read_sas("<FileName>.sas7bdat")  # 读取 SAS 的数据文件 (.sas7bdat 格式)

# --- 快速导入表格（纯文本文件）---
library(readr) # 加载 `readr` 包
df_csv <- read_csv("<FileName>.csv") # 读取逗号分隔值文件 (.csv)
df_table <- read_table("<FileName>.txt") # 读取用空格或制表符分隔的表格文件

# --- 导入 Excel 表格 ---
library(readxl) # 加载 `readxl` 包
df_excel <- read_excel("<FileName>.xls") # 读取.xls 格式文件
df_excel2 <- read_excel("<FileName>.xlsx") # 读取.xlsx 格式文件

# --- 导入 JSON 数据 ---
library(rjson) # 加载 `rjson` 包
df_json <- fromJSON(file = "<FileName>.json") # 从一个 .json 文件中解析数据

# --- 导入 XML / HTML 数据 ---
library(XML) # 加载 `XML` 包
df_xml <- xmlTreeParse("<url>") # 解析 XML 文件
df_html <- readHTMLTable("<url>", which = 3) # 提取页面上第 3 个表格
  • 万能读取：rio::import()
  • 优点：自动识别格式
  • 缺点：速度略慢；大数据建议用专门包

# 使用 rio 包的 import 函数，自动识别格式并导入
library(rio)
df_anything <- import(<AnyTypeOfData>)

14 输出

  • 基础语法：<函数名>(<储存对象>, <储存路径>, <其他条件>)
  • 函数名常以 save、write 开头

15 储存为R数据

  • .rds：单一对象；.rdata：多对象工作空间

# 保存为 RDS 单一对象
saveRDS(df_toy, file = "df_toy.rds")

# 保存为 RData 工作空间
save(df_toy, ls_monks, file = "./data/test.rdata")
  • 推荐 .rds：调取/整理更清晰

16 储存为csv文件

  • 通用性强；用 write.csv()

# 将 df_toy 保存为 CSV 文件
write.csv(df_toy, file = "toy.csv")

CSV与中文数据

  • 中文数据存储时选 UTF-8 编码，避免乱码

17 储存为其他格式

  • 可存为 STATA、SPSS、SAS、Excel、JSON、Matlab、HTML 等
  • 万能输出：rio::export()，根据后缀自动识别

数据储存效率

  • 中小规模数据：RData 足矣
  • 大数据：可考虑 qs、fst 等特殊格式
  • 兼容性/稳定性需权衡
